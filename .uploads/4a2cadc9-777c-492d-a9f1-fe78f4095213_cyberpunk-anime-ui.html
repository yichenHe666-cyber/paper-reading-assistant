好的！我来为你深入分析这篇开创性的论文。

核心贡献：完全基于注意力机制，抛弃了 RNN/CNN 的序列建模范式。

自注意力机制的核心公式：

$$\text{Attention}(Q, K, V) = \text{softmax}\!\left(\frac{QK^T}{\sqrt{d_k}}\right) V$$

其中 $Q$、$K$、$V$ 分别是 Query、Key、Value 矩阵，$d_k$ 是 Key 的维度。除以 $\sqrt{d_k}$ 是为了防止点积过大导致 softmax 梯度消失。

🔧 search_paper — 检索到 3 篇相关论文

多头注意力（Multi-Head Attention）将 Q、K、V 分别映射到 $h$ 个子空间：

$$\text{MultiHead}(Q,K,V) = \text{Concat}(\text{head}_1, \ldots, \text{head}_h)\,W^O$$

每个 head 使用不同的投影矩阵，使模型能关注不同位置的不同表示子空间。

PyTorch 实现核心代码：

python
import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class ScaledDotProductAttention(nn.Module):
    def __init__(self, d_k):
        super().__init__()
        self.d_k = d_k

    def forward(self, Q, K, V, mask=None):
        scores = torch.matmul(Q, K.transpose(-2, -1))
        scores = scores / math.sqrt(self.d_k)
        if mask is not None:
            scores = scores.masked_fill(mask == 0, -1e9)
        attn = F.softmax(scores, dim=-1)
        return torch.matmul(attn, V)

位置编码（Positional Encoding）为什么用正弦函数？和可学习位置编码相比有什么优劣？
14:35 · 42 tokens
🐂

论文使用正弦位置编码的原因：

$$\text{PE}(pos, 2i) = \sin\!\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$

$$\text{PE}(pos, 2i+1) = \cos\!\left(\frac{pos}{10000^{2i/d_{\text{model}}}}\right)$$

优势：允许模型外推到训练时未见过更长序列长度（虽然实际效果有限）。正弦函数的线性变换特性使得模型可以学习相对位置关系。

与可学习位置编码对比：现代实践中（如 GPT、BERT），可学习位置编码更常用，因为它更灵活。但正弦编码无需训练参数，泛化性理论上更好。
